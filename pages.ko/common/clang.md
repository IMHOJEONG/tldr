# clang

> C, C++, Objective-C 소스 파일 컴파일. GCC의 대체 컴파일러로 사용 가능.
> LLVM의 일부.
> 더 많은 정보: <https://clang.llvm.org/docs/ClangCommandLineReference.html>.

- 여러 소스 파일을 컴파일하여 실행 파일을 생성:

`clang {{경로/대상/소스파일1.c 경로/대상/소스파일2.c ...}} {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- 모든 오류와 경고 메시지 출력을 활성화:

`clang {{경로/대상/소스파일.c}} -Wall {{[-o|--output]}} {{출력_실행파일}}`

- 일반적인 경고를 표시, 디버그 심볼을 포함, 디버깅에 영향을 주지 않는 최적화를 수행:

`clang {{경로/대상/소스파일.c}} -Wall {{[-g|--debug]}} -Og {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- 다른 경로에 있는 라이브러리를 포함하여 컴파일:

`clang {{경로/대상/소스파일.c}} {{[-o|--output]}} {{경로/대상/출력_실행파일}} -I{{경로/대상/헤더}} -L{{경로/대상/라이브러리}} -l{{library_name}}`

- 소스 코드를 LLVM 중간 표현 (IR)으로 컴파일:

`clang {{[-S|--assemble]}} -emit-llvm {{경로/대상/소스파일.c}} {{[-o|--output]}} {{경로/대상/출력.ll}}`

- 링크과정 없이 소스 코드를 오브젝트 파일로 컴파일:

`clang {{[-c|--compile]}} {{경로/대상/소스파일.c}}`

- 성능을 위해 컴파일된 프로그램을 최적화:

`clang {{경로/대상/소스파일.c}} -O{{1|2|3|fast}} {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- 버전 표시:

`clang --version`
